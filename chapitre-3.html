Chapitre 3 — Switching & LAN

Switch · VLANs · 802.1Q · STP/RSTP · LACP · Segmentation réseau

1. Le switch

Un switch opère en couche 2 — il lit les adresses MAC et décide sur quel port envoyer une trame. Il maintient une table MAC apprise automatiquement en observant le trafic.

Switch Couche 2

Lit les adresses MAC · connecte des machines dans le même réseau · transparent pour les couches supérieures · la trame n'est pas modifiée

Routeur Couche 3

Lit les adresses IP · connecte des réseaux différents · intervient activement (remplace les MAC, décide du chemin)

Même subnet → switch suffit, communication directe via MAC Subnets diff. → routeur obligatoire, communication via IP
Le switch est transparent — du point de vue des couches 3+, PC A et PC B communiquent "directement". Le switch ne modifie rien, il relaie juste la trame au bon port.

Switch non manageable vs manageable

Non manageable

Plug and play. Aucune interface de gestion. Pas de VLANs, pas de STP configurable. Adapté aux très petits réseaux.

Manageable

Accessible via port console (câble RJ45/USB), SSH/Telnet (IP de gestion) ou interface web HTTPS. Supporte VLANs, STP, LACP.

2. Les VLANs

Un VLAN découpe un switch physique en plusieurs switches virtuels isolés. Chaque port se voit attribuer un numéro de VLAN (1 à 4094). Les ports avec le même numéro peuvent se parler — les ports avec des numéros différents sont complètement isolés.

Pourquoi 4094 ? Le tag 802.1Q utilise 12 bits pour le numéro de VLAN → 2¹² = 4096 combinaisons. VLAN 0 et VLAN 4095 sont réservés → 4094 utilisables.

Les deux types de ports

Port access

Connecté à une machine finale (PC, imprimante, caméra). Appartient à un seul VLAN. La machine ne sait pas qu'un VLAN existe — totalement transparent.

Port trunk

Connecté à un autre switch ou routeur. Transporte plusieurs VLANs sur un seul câble grâce aux tags 802.1Q. Utilisé pour relier switches entre eux.

Le tag 802.1Q

Sur un port trunk, chaque trame reçoit un tag de 4 octets indiquant son VLAN. Ce tag est ajouté par le switch à l'entrée et retiré à la sortie — les machines finales ne le voient jamais.

Trame normale : [ETH] [IP] [TCP] [Données] Trame taguée : [ETH] [802.1Q — VLAN 10] [IP] [TCP] [Données]
Pour communiquer entre deux VLANs différents, le trafic doit obligatoirement passer par un routeur. Le switch seul ne peut pas router entre VLANs.

3. Le STP — Spanning Tree Protocol

Quand plusieurs switches sont interconnectés avec redondance, des boucles se forment. Un broadcast tourne en boucle infiniment entre les switches — c'est la tempête de broadcast. Le réseau est saturé en millisecondes.

STP résout ce problème en bloquant volontairement certains liens pour éliminer les boucles, tout en gardant la redondance disponible en cas de panne.

Fonctionnement

1. Élection du Root Bridge

Les switches s'échangent des messages BPDU (Bridge Protocol Data Unit — heartbeats envoyés toutes les 2 secondes). Le switch avec la priorité la plus basse est élu Root Bridge. Si priorité égale, la MAC la plus basse départage.

En pratique on configure manuellement la priorité du switch core à 4096 (vs 32768 par défaut) pour forcer son élection.

2. Calcul de l'arbre

Chaque switch calcule son chemin le plus court vers le Root Bridge selon les coûts de liens. Les liens redondants sont mis en état bloqué.

3. Détection de panne

Si un switch cesse de recevoir des BPDUs d'un voisin, il déclare le lien mort et réactive automatiquement le port bloqué. Pas d'intervention humaine.

États d'un port STP

Forwarding

Actif — le trafic passe normalement.

Blocking

Bloqué — aucun trafic. Écoute les BPDUs pour détecter une panne.

Learning

Transitoire — apprend les MAC sans encore faire passer le trafic.

STP vs RSTP

STP classique 1985

Passif — attend 20s sans BPDU avant de déclarer une panne, puis passe par Listening (15s) + Learning (15s).

Convergence : 30 à 50 secondes
Inacceptable pour VoIP, streaming, apps critiques.

RSTP 802.1w — 2001

Actif — négociation directe entre switches. Le voisin répond immédiatement quand un lien tombe. Port alternatif déjà en attente.

Convergence : moins d'une seconde
Standard utilisé partout aujourd'hui.

4. L'agrégation de liens — LACP

Sans LACP, STP bloque les liens redondants — la bande passante de ces liens est perdue. LACP fusionne plusieurs liens physiques en un seul lien logique. STP voit ce lien logique comme un seul lien — pas de boucle, pas de blocage. Tous les liens physiques sont actifs simultanément.

Sans LACP : Switch A ←actif 1G→ Switch B + ←bloqué 1G→ = 1 Gbps réel Avec LACP : Switch A ←══════ LAG 2 Gbps ══════→ Switch B = 2 Gbps réel

Bande passante additive

Les capacités des liens physiques s'additionnent. 2 × 1 Gbps = 2 Gbps de capacité totale pour l'ensemble des flux.

Un flux unique ne dépasse pas 1 Gbps — la répartition se fait par flux via un algorithme de hachage (MAC ou IP src/dst).

Redondance automatique

Si un lien physique tombe, LACP bascule immédiatement tout le trafic sur les liens restants. Pas de coupure, pas d'intervention humaine.

La bande passante diminue le temps de la réparation, mais le réseau reste fonctionnel.

LACP statique vs dynamique

Statique

Config manuelle des deux côtés. Simple mais risqué — une incohérence peut créer des boucles.

LACP dynamique Recommandé

Négociation automatique (802.3ad). Refuse de former le groupe si config incohérente. Évite les problèmes.
